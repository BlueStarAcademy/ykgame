## 1. 제안 개요

### 1.1 제안 배경

YK건기는 YANMAR, JOHN DEERE, MANITOU, WIRTGEN, VÖGELE, GEHL, HAMM, KLEEMANN 등 **8대 글로벌 장비 브랜드**를 취급하고 있습니다. 잠재 고객·내부 구성원·전시·행사 현장에서 브랜드를 **체험형 콘텐츠**로 전달할 필요가 있으며, 별도 앱 설치 없이 **모바일 브라우저**에서 즉시 플레이 가능한 경량 게임 플랫폼이 효과적인 마케팅·교육 수단이 될 수 있습니다.

### 1.2 제안 목적

본 제안은 YK건기 8대 브랜드를 테마로 한 **모바일 웹 캐주얼 미니게임 플랫폼**을 기획·설계·개발·배포하는 것을 목적으로 합니다. 사용자는 1~2분 분량의 미션을 플레이하고, **점수·별(재화)·월간 랭킹**을 통해 반복 참여를 유도하며, 관리자는 **회원·재화·활성 상태**를 통합 관리할 수 있습니다.

### 1.3 제안 핵심 가치

| 가치 | 설명 |
|------|------|
| **브랜드 체험** | 8종 장비 브랜드별 고유 미션으로 제품군 인지도 제고 |
| **접근성** | 앱 설치 불필요, QR·URL로 즉시 접속 가능한 모바일 웹 |
| **참여 유도** | 점수·별·월간 랭킹으로 재방문·경쟁 요소 제공 |
| **운영 효율** | 관리자 패널을 통한 사용자·재화 일괄 관리 |
| **확장성** | 공통 게임 엔진 기반으로 신규 브랜드·미션 추가 용이 |

## 2. 요구사항 이해 및 범위 정의

### 2.1 사업 목표

1. YK건기 취급 **8대 브랜드**를 각 1종의 캐주얼 미니게임으로 구현
2. **회원가입·로그인·닉네임** 기반 개인화 플레이 환경 제공
3. 플레이 결과 **점수 저장·별(재화) 지급·월간 랭킹** 노출
4. **관리자**가 사용자·재화를 웹에서 관리
5. **클라우드 자동 배포**로 운영 환경 구축

### 2.2 대상 사용자

| 구분 | 설명 |
|------|------|
| **일반 사용자(USER)** | 모바일 웹으로 게임 플레이, 랭킹 확인 |
| **관리자(ADMIN)** | 사용자 조회·검색, 재화 지급/차감, 계정 활성 관리 |

## 3. 기술 제안

### 3.1 기술 스택

```
┌─────────────────────────────────────────────────┐
│  모바일 브라우저 (iOS Safari / Android Chrome)   │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│  Next.js 16 (App Router) + React 19 + Tailwind 4 │
│  Phaser 4 (클라이언트 게임 엔진)                  │
└─────────────────────┬───────────────────────────┘
                      │ REST API
┌─────────────────────▼───────────────────────────┐
│  Next.js API Routes + NextAuth v5 (JWT)          │
│  Prisma 7 + PostgreSQL                           │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│  Railway (호스팅 + Postgres + CI/CD)             │
└─────────────────────────────────────────────────┘
```

### 3.2 기술 선정 근거

| 기술 | 선정 이유 |
|------|-----------|
| **Next.js** | SSR/라우팅·API 통합, 빠른 MVP 구축, Vercel/Railway 호환 |
| **Phaser 4** | 2D 캐주얼 게임에 검증된 엔진, 터치·타이머·Scene 재사용 |
| **PostgreSQL** | 랭킹·집계 쿼리 안정성, Railway managed DB |
| **NextAuth** | Credentials 인증, JWT 세션, OAuth 확장 가능 |
| **Railway** | GitHub 연동 자동 배포, Postgres 통합, 운영 부담 최소화 |

### 3.3 게임 아키텍처

6종 공통 Scene 클래스로 8개 게임을 효율적으로 구현합니다.

| Scene 타입 | 적용 게임 | 특징 |
|------------|-----------|------|
| Collect | YANMAR | 수집·운반 |
| Drive | JOHN DEERE, WIRTGEN | 주행·경로 |
| Deliver | MANITOU, GEHL | 픽업·배달 |
| Pave | VÖGELE | 전진·폭 조절 |
| Compact | HAMM | 전진·진동 |
| Sort | KLEEMANN | 분류·선별 |

**점수 공식**: `점수 = 진행률 × 10 + 남은 시간(초) × 2`  
**별 등급**: 100% → 3★ / 80% → 2★ / 50% → 1★
