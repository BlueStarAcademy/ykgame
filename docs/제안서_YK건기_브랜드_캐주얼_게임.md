## 요약 (Executive Summary)

YK건기가 취급하는 8대 글로벌 장비 브랜드를 주제로, **모바일 웹에서 즉시 즐길 수 있는 캐주얼 미니게임 플랫폼**을 제안합니다.

별도 앱 설치 없이 QR 코드 또는 URL로 접속하면, 사용자는 1~2분 분량의 브랜드별 미션을 플레이하고 점수·별(재화)·월간 랭킹을 통해 반복 참여할 수 있습니다. 관리자는 웹 기반 패널에서 회원과 재화를 통합 운영합니다.

본 제안은 **브랜드 체험·마케팅·내부 교육**을 하나의 디지털 채널로 연결하고, 향후 소셜 로그인·장비 업그레이드·고품질 게임 에셋 등으로 단계적 확장이 가능한 구조를 목표로 합니다.

## 1. 제안 배경

### 1.1 현황 및 과제

YK건기는 YANMAR, JOHN DEERE, MANITOU, WIRTGEN, VÖGELE, GEHL, HAMM, KLEEMANN 등 **8대 장비 브랜드**를 취급하고 있습니다. 각 브랜드는 제품군·적용 분야·고객층이 다르며, 전시회·내방·행사·SNS 등 다양한 접점에서 **짧고 기억에 남는 브랜드 경험**이 필요합니다.

기존 브로슈어·영상 중심의 홍보는 정보 전달에는 유효하나, **직접 체험·참여·경쟁**을 유발하는 데는 한계가 있습니다. 특히 모바일 환경에서 앱 설치 부담 없이 즉시 참여할 수 있는 콘텐츠는 전시·이벤트·온라인 캠페인 모두에서 활용 가치가 높습니다.

### 1.2 제안 방향

이에 본 제안은 다음과 같은 방향으로 플랫폼을 구축합니다.

- **8대 브랜드 = 8종 미니게임** : 브랜드별 장비 특성을 반영한 1~2분 미션
- **모바일 웹 우선** : iOS·Android 브라우저에서 터치 조작으로 즉시 플레이
- **참여·재방문 유도** : 점수, 별(재화), 월간 랭킹으로 동기 부여
- **운영 편의** : 관리자 웹에서 회원·재화·활성 상태 관리
- **클라우드 자동 배포** : GitHub 연동 CI/CD로 안정적 운영

## 2. 제안 목적 및 기대 효과

### 2.1 제안 목적

| 목적 | 설명 |
|------|------|
| 브랜드 인지 | 8대 브랜드를 게임 미션으로 자연스럽게 체험 |
| 참여 확대 | 점수·랭킹·재화로 재방문·공유 유도 |
| 운영 효율 | 관리자 도구로 사용자·이벤트 재화 일괄 관리 |
| 확장 기반 | 공통 게임 엔진으로 신규 브랜드·미션 추가 용이 |

### 2.2 기대 효과

**마케팅**  
전시·행사·SNS에서 QR 하나로 8브랜드를 동시에 체험할 수 있습니다. 짧은 플레이 시간으로 부스 체류·참여율을 높일 수 있습니다.

**교육**  
신입·영업·서비스 직원이 브랜드별 장비 용도(굴삭·농기·물류·도로·포장·축산·다짐·분쇄 등)를 게임을 통해 학습할 수 있습니다.

**고객 관계**  
경량 캐주얼 게임으로 브랜드 친숙도와 기억도를 높이고, 향후 이벤트·프로모션과 연계할 수 있습니다.

**데이터 활용**  
플레이 횟수·점수·랭킹 데이터를 바탕으로 인기 브랜드·참여도를 파악할 수 있습니다. (통계 대시보드는 선택 확장 항목)

## 3. 서비스 개요

### 3.1 서비스명

**YK건기 브랜드 캐주얼 게임** (가칭: ykgame)

### 3.2 서비스 구성

```
[사용자]                         [관리자]
  │                                 │
  ├─ 회원가입·로그인·닉네임          ├─ 사용자 조회·검색
  ├─ 홈 (8종 게임 선택)              ├─ 재화 개별·일괄 지급
  ├─ 미니게임 플레이                 └─ 계정 활성/비활성
  ├─ 점수·별 획득
  └─ 월간 랭킹 확인
```

### 3.3 사용자 여정

1. **가입·로그인** : 아이디·이메일·비밀번호로 회원가입 후 로그인
2. **닉네임 설정** : 최초 1회 게임 표시명 설정 (2~12자)
3. **게임 선택** : 홈 화면에서 8종 브랜드 게임 중 선택
4. **미션 플레이** : 약 90초 내 목표 달성 (터치 조작)
5. **결과 확인** : 점수·별·월간 Top 10 랭킹 확인
6. **재도전·홈** : 재시도 또는 다른 브랜드 게임 이동

| 점수·재화 | 플레이 종료 시 자동 저장, 진행률에 따라 0~3별 지급 |
| 월간 랭킹 | 게임별 이번 달 Top 10 (결과 화면) |

### 4.2 미니게임 8종

| 번호 | 브랜드 | 미션 | 조작 | 목표 |
|:----:|--------|------|------|------|
| 1 | YANMAR (얀마) | 굴삭기로 흙 옮기기 | 방향키 + 작업 | 10개 수집 |
| 2 | JOHN DEERE (존디어) | 트랙터로 밭 갈기 | 조향 + 전진 | 16 lap |
| 3 | MANITOU (마니또) | 톤백 옮기기 | 이동 + 리프트 | 5회 배달 |
| 4 | WIRTGEN (빌트겐) | 도로 작업하기 | 조향 + 전진 | 20 lap |
| 5 | VÖGELE (보겔) | 아스팔트 포장 | 전진 + 폭 조절 | 30 진행 |
| 6 | GEHL (겔) | 축사 퇴비 옮기기 | 이동 + 리프트 | 6회 배달 |
| 7 | HAMM (햄) | 도로 다지기 | 전진 + 진동 | 35 진행 |
| 8 | KLEEMANN (클래만) | 암석 분류 | 대·중·소 버튼 | 12개 분류 |

**점수 산정**  
점수 = 진행률 × 10 + 남은 시간(초) × 2

**별 등급**  
100% 달성 → 3별 / 80% → 2별 / 50% → 1별

## 5. 기술 제안

### 5.1 시스템 구성

| 계층 | 기술 | 역할 |
|------|------|------|
| 프론트엔드 | Next.js 16, React 19, Tailwind CSS 4 | 화면·라우팅·모바일 UI |
| 게임 | Phaser 4 | 2D 캐주얼 미니게임 엔진 |
| 백엔드 | Next.js API Routes | REST API |
| 인증 | NextAuth v5 (JWT) | 로그인·세션·역할(USER/ADMIN) |
| 데이터베이스 | PostgreSQL + Prisma 7 | 사용자·점수·랭킹 저장 |
| 배포 | Railway | 호스팅·DB·GitHub 자동 배포 |

### 5.2 기술 선정 이유

- **Next.js** : 웹 UI와 API를 하나의 프로젝트에서 관리하여 개발·배포 효율이 높습니다.
- **Phaser** : 2D 캐주얼 게임에 검증된 엔진으로, 터치 조작·타이머·Scene 재사용이 용이합니다.
- **PostgreSQL** : 랭킹·집계 쿼리에 안정적이며, Railway managed DB와 연동됩니다.
- **Railway** : GitHub push 시 자동 빌드·마이그레이션·배포가 가능해 운영 부담을 줄입니다.

### 5.3 게임 개발 구조

8개 게임은 6종 공통 Scene 클래스를 재사용하여 효율적으로 개발합니다.

| Scene 유형 | 적용 브랜드 | 미션 특성 |
|------------|-------------|-----------|
| Collect (수집) | YANMAR | 수집·운반 |
| Drive (주행) | JOHN DEERE, WIRTGEN | 경로 주행 |
| Deliver (배달) | MANITOU, GEHL | 픽업·배달 |
| Pave (포장) | VÖGELE | 전진·폭 조절 |
| Compact (다짐) | HAMM | 전진·진동 |
| Sort (분류) | KLEEMANN | 크기별 분류 |

신규 브랜드 추가 시 기존 Scene을 확장·재사용할 수 있어 유지보수 비용을 절감할 수 있습니다.
